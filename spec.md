# 개인용 퀴즈 플랫폼 스펙

## 목적

- Quizlet과 유사한 **암기/자가 점검** 경험을 로컬에서 사용한다.
- **실서비스·다중 사용자 대상이 아님.** 본인 전용, 브라우저 로컬 저장 중심.
- 회사 등에서 **눈에 띄지 않도록** UI는 최소한의 정보·중립적인 톤으로 유지한다.

## 기술 스택

- **Vite + React + TypeScript**

## 데이터 모델 (JSON)

- 최상위 객체는 `version`(숫자)과 `sections`(배열)를 가진다.
- 각 **섹션(단원)** 은 다음을 가진다.
  - `id`: 문자열 (고유 식별)
  - `title`: 표시용 제목
  - `items`: 문제 배열
- 각 **문항** 은 다음을 가진다.
  - `question`: 질문 문자열
  - `answer`: 정답 문자열 (필수)
  - `answers`: (선택) 복수 정답·동의어 허용 시 문자열 배열. 채점 시 `answer`와 함께 후보로 취급한다.

예시 구조는 저장소의 `public/example-bank.json` 또는 앱 내 기본 샘플을 참고한다.

## 기능 요구사항

### 1. JSON으로 문제·답 관리

- 텍스트 영역에서 JSON을 직접 편집하고 **적용(저장)** 할 수 있다.
- **파일 가져오기**(JSON), **파일 내보내기**로 백업·이동이 가능하다.
- 유효하지 않은 JSON/스키마는 오류 메시지로 알린다.
- 브라우저 **localStorage**에 마지막으로 적용한 뱅크를 유지한다 (새로고침 후에도 유지).

### 2. 섹션(단원)별 구분

- 섹션 단위로 문제를 묶는다.
- 학습 화면에서 **섹션을 선택**한 뒤 해당 섹션만 사용한다.

### 3. 학습(퀴즈) 동작

- 선택한 섹션의 문항 순서는 **무작위(셔플)** 로 섞인다.
- 한 번에 한 문항의 **질문**을 보여 주고, 사용자가 **답을 입력**한다.
- 채점은 공백 트림 후 **대소문자 무시** 문자열 비교. `answers`가 있으면 그 중 하나와 일치하면 정답.
- 정답/오답 피드백 후 **다음**으로 진행. 세트 종료 시 간단한 결과(맞은 개수 등)를 볼 수 있다.

### 4. UI 방침

- **심플·밀도 낮음·중립 색** (문서/내부 도구처럼 보이도록).
- 탭 또는 구역으로 **「학습」** 과 **「데이터」** 를 분리해 학습 화면은 최소 요소만 노출한다.

## 비범위 (의도적으로 하지 않음)

- 서버·계정·동기화
- 공개 공유·리더보드
- 풍부한 미디어 카드 UI (이미지/음성 등)

## 변경 시 이 문서

- 동작이나 데이터 형식이 바뀌면 이 파일을 함께 갱신해, 이후 세션에서 컨텍스트 파악이 쉽도록 한다.
